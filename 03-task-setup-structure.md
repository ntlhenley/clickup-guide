# Task Setup Structure

## Parent/Subtask Structure Requirements
- Clearly define parent tasks and their corresponding subtasks.
- Each parent task should have specific goals that are measurable.
- Subtasks should outline distinct steps to achieve the parent task.

## Recurring Task Setup
- Use a clear naming convention to indicate recurring tasks (e.g., "Monthly Report - Recurring").
- Specify the frequency (daily, weekly, monthly) in the task description.

## One-Time Task Setup
- One-time tasks should have specific deadlines.
- Indicate clearly in the task name that it is a one-time task (e.g., "Q1 Financial Review - One Time").

## Naming Conventions
- Use concise, descriptive names that include the type of task (Recurring/One-Time).
- Follow a format: `[Type] - [Description]` (e.g., "[Recurring] - Weekly Team Meeting").

## Placement Guidelines
- Place tasks in the correct section of the project according to their priority and type.
- Use labels or tags to categorize tasks (e.g., Urgent, Important, Routine).

## Timing Standards
- Tasks should have estimated completion times listed (in hours or days).
- Use time tracking tools where applicable to monitor time spent on tasks.

## What Happens When a Task is Complete
- Mark the task as complete in the tracking system.
- Ensure all subtasks are also marked complete before closing the parent task.
- Provide brief insights or notes on the task completion in the comments section.
